VOLABIT API DOCUMENTATION

The following document is divided into three parts. Part I describes how to gain access to the swagger documentation. Part II describes how to generate your first API token and how to refresh your token. Finally, Part III covers the API calls that can be made as well as some tips for implementation.

PART I. ACCESS SWAGGER DOCUMENTATION

The swagger documentation provides an overview of all the methods with their parameters and expected responses.

  1. Go to https://www.volabit.com/api-docs/
  2. Enter the following APP ID: a47f325d0718ccd3e8aa1ec29c47d6d3a2cf18bca639b3fb1014b0362d163d06 and click Save.

  3. Create an account or sign in to an existing account and click on the Authorize button.

    To get full access to the API we will require you to send us some KYC/AML documents. Please contact support@volabit.com for further instructions.

PART II. CREATING AN APP AND MANAGING TOKENS

Once the paperwork has been cleared, you can generate an Application ID with its secret key and use these credentials to generate a token and all subsequent token refreshes.

Go to https://www.volabit.com/oauth/applications and create an application with the default Redirect URL urn:ietf:wg:oauth:2.0:oob. In the following view, click Authorize and follow instructions until you are provided with an authorization code. This authorization code (AUTH_CODE) will be used to generate an access_token and a refresh_token with the following call:

    curl -F grant_type=authorization_code \
    -F client_id=APP_ID \
    -F client_secret=APP_SECRET \
    -F code=AUTH_CODE \
    -F redirect_uri=urn:ietf:wg:oauth:2.0:oob \
    -X POST https://www.volabit.com/oauth/token

Tokens can be used to make calls until they expire in 8 hours. Afterwards you will have to use refresh_token to generate a new token.

    curl -F grant_type=refresh_token \
    -F client_id=APP_ID \
    -F client_secret=APP_SECRET \
    -F refresh_token=REFRESH_TOKEN \
    -X POST https://www.volabit.com/oauth/token

How to check your token info:

    curl -H "Authorization: Bearer YOUR_TOKEN" \
      https://www.volabit.com/oauth/token/info

Tip: Check for an expired token before making each call

Now that you can create and refresh tokens, lets move on to the interesting part.

PART III: API IMPLEMENTATION

For services partnering with Volabit

Overview

  • Send money to a current user in Volabit, or to anyone in Mexico that can sign up for Volabit.

  • Request money from an existing user.

    Required Endpoints:

  • api/v1/users [POST]
  • api/v1/users/send [POST]
  • api/v1/users/requests [POST]
  • api/v1/users/requests/methods [POST]

Create User api/v1/users [POST]

For a seamless money sending and receiving experience, we require that you to register your customer as a user on Volabit.com.

After you have created a user you can proceed to push and request payments from him/her.

Send Money to a User api/v1/users/send [POST]

In order to minimize exposure to fluctuations in the value of Bitcoins, approved services can generate green addresses for instant verification. Any money sent to one of these trusted addresses will be immediately converted to fiat currency (prior to any confirmations).

Request Payment from a User api/v1/users/requests [POST]

You will create a payment slip for the user and will provide a Bitcoin address for the payment. Whenever the user pays the slip (via convenience store deposits, bank deposit, etc) the deposited amount will automatically be converted to Bitcoins and immediately sent to the address that was provided when making the API call.

Send Money to User Implementation

  1. If you haven’t already, create a Volabit account for your user. Ideally you will use the same email address that he used to sign up for your service. This action is done with a POST api/v1/users with the following parameters:

    • [email] required: email of your user
    • [password] optional: if left blank Volabit will generate a random password for the user. An email notification will be sent to the user.

    Bash Example

      curl -H "Authorization: Bearer YOUR_TOKEN" \
    -F user[email]=a_user@volabit.com \
    -X POST https://www.volabit.com/api/v1/users

  2. Get a quote in BTC for the amount of Mexican Pesos to be sent to the user. GET api/v1/spot-prices allows for BTC/MXN and MXN/BTC conversions that include Volabit’s fees. *All BTC amounts are denominated in Satoshis.

    • [currency_from] required: Base currency you are converting from
    • [currency_to] required: Counter currency you are converting to.
    • [amount] required: Amount of the Base currency you want to convert to Counter currency. For sending 10,000 pesos to a Volabit user the params are: currency_from = MXN, currency_to = BTC, amount = 1000000

    Bash Example

      curl -F currency_from=BTC \
    -F currency_to=MXN \
    -F amount=100000000 \
    https://www.volabit.com/api/v1/spot-prices

  3. Generate a Bitcoin address for the user via POST api/v1/users/send. Any bitcoins sent to this address will be immediately converted to fiat with 0 confirmations. Note that this method allows you to select who will pay the transaction fee. If Sender is selected, the recipient will receive the full amount entered. If Receiver is selected, the fees will be deducted from the amount sent. The parameters are:

    • [email_to] required: Email of the Volabit user that will be credited the money.
    • [currency_from] required: Only BTC for the moment
    • [currency_to] required: Only MXN for the moment
    • [amount] required: Amount in fiat (MXN) in cents the Volabit user will receive.
    • [fee_player] required: Will the fee be deducted from the Volabit user or will the fee be included in the Bitcoin amount sent (sender pays the fee)?

    Bash Example

      curl -H "Authorization: Bearer YOUR_TOKEN" \
    -F email_to=a_user@volabit.com \
    -F currency_from=BTC \
    -F currency_to=MXN \
    -F amount=500000 \
    -F fee_player=SENDER
    https://www.volabit.com/api/v1/users/send

Request Payment from User Implementation

  1. If you haven’t already, create a Volabit account for your user. Ideally you will use the same email address that he used to sign up for your service. This action is done with a POST api/v1/users with the following parameters:

    • [email] required: email of your user
    • [password] optional: if left blank Volabit will generate a random password for the user. An email notification will be sent to the user.

    Bash Example

      curl -H "Authorization: Bearer YOUR_TOKEN" \
    -F user[email]=a_user@volabit.com \
    -X POST https://www.volabit.com/api/v1/users

  2. Create a payment slip for your user so that you can collect a payment from him/her. POST api/v1/users/requests

    • [email] required: Email of the Volabit user a payment is being requested from
    • [currency] required: The payment slip’s currency (only MXN for now)
    • [amount] required: The amount for which the payment slip will be created (in cents).
    • [type] required: Method of payment for the payment slip (e.g. Convenience store, bank branch, bank transfer) You can view all accepted methods through GET api/v1/users/requests/methods.
    • [address] required: Bitcoin address where you would like your payment to be sent. Bitcoins will be sent the moment the fiat payment is confirmed in Volabit’s platform.
      curl -H "Authorization: Bearer YOUR_TOKEN" \
    -F email=a_user@volabit.com \
    -F currency=MXN \
    -F amount=500000 \
    -F type=oxxo_mx \
    -F address=q15YTN2uxdJ7GMkvQ2prVJVfHcFw3t5YEwjq \
    https://www.volabit.com/api/v1/users/requests
